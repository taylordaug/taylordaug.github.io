Top of the Class

February 3rd, 2016

Once I graduate from DevBootcamp and get hired, I am rewarding myself by getting a kitten.

However, I know I can be very indecisive, so with my new knowledge of Ruby classes, I am going to create a program that will randomly generate the type of cat I should get. I want to be able to input options of breeds and names that I love, and also include whether it should be a male or female. Each new cat, or instance, that is created from this class will tell me the breed, gender, and name of my new cat. I'll even be able to pass my cat generator on to friends and it will continue to spit out cats, or objects, that follow this same blueprint of breed, name and gender. The name of this class with be 

YourFutureCat
. I need to set the blueprint that I want all cats created from this class to follow. This is called initializing and looks like this

class YourFutureCat
def initialize(breeds, gender, girl_names, boy_names)
    @breed = breeds
    @gender = gender
    @girl_names = girl_names
    @boy_names = boy_names
  end

See those @ signs? Those are storing the options for breeds, gender, and names in variables that can be used across methods in this class. This means that I can now call the variable @breed in the next method that I define in the class: 

def breed
    @breed.sample
  end

The variable @breed was created in the method 

 initialize
, but I can still use it in the new method 
breed
. Usually, you variables live inside the methods they are created in. But @instancevariables work across the instance of the class, or this particular object you are creating with the class.
